MACHAMP Lab · Jan 2026 – Present

Scalar-on-Tensor Regression with Unbalanced Tensor Predictors

AwardExcellence in Research · UMD Undergraduate Research Day
RoleUndergraduate Researcher
MethodsTensor regression, clustering, simulation studies

Overview

I developed TEPIG (TEnsor sparse grouP lasso with clusterInG), a method that predicts kidney function one year after transplant from 43 features across multi-slide biopsies, under the guidance of Professor Jeremy Rubin. The work won the Excellence in Research Award among 550+ participants at UMD's Undergraduate Research Day.

The project extends CLUSSO to better handle pathology images where each patient can have a different number of tissue structures. I use clustering and weighting to turn those variable slides into a consistent format for modeling while keeping the output interpretable.

Methods and Evaluation

  • Automated GMM clustering, tensor construction, and regression across 683K tubule observations with an end-to-end Python pipeline, cutting manual data processing time by ~80%.
  • Handles unbalanced tensor predictors — a different number of tissue structures per patient — via clustering and weighting, a case prior methods like CLUSSO handle poorly.
  • Validated across 200-repetition simulations in 70 settings on SLURM-managed HPC: over 90% fewer false positives than existing methods, at a true-positive rate of ≈1.0.
Python NumPy scikit-learn SLURM HPC

Independent ML Research · 2024 – 2025

Forecasting Soccer Player Value via Integration of Injury History with Machine Learning

GitHub ↗ Research Paper ↗
TypeIndependent ML research
DatasetRoughly 17K player-seasons
Best ModelR² 0.856 · MAE ~$3.82M

Overview

I built a time-aware machine learning pipeline to forecast soccer player transfer value by integrating multi-year injury history, season context, and more than 300 engineered features across roughly 17K player-seasons.

The modeling setup uses 1- to 5-year sliding windows so each horizon evaluates how well value can be predicted as a player's career evolves, rather than assuming seasons are independent.

Modeling Approach

  • Engineered 300+ features from player stats, multi-year injury history, and season context scraped from Transfermarkt, across roughly 17K player-seasons.
  • Trained XGBoost against baseline regressors over 1–5-year sliding windows, using temporal validation to prevent leakage across seasons — best model reached R² 0.856 with MAE ≈ $3.82M on an 80/20 split.
  • Authored a 20-page research paper documenting the feature engineering, validation design, and model comparisons.
Python R XGBoost pandas matplotlib
